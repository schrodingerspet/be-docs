<Callout type="insight">
**Why is this a breakthrough?** The sub-200ms latency is the true breakthrough here. A 99% accurate model is useless if the drone crashes before the model finishes inference. Sophia S's pipeline emphasizes real-time viability over marginal accuracy gains.
</Callout>

<Accordion title="Understanding the Trade-off">
Achieving sub-200ms latency required aggressive dimensionality reduction using PCA. This sacrifices some interpretability (unlike the SHAP method in Paper 1) but ensures the drone's flight controller can receive the "abort mission" or "ignore sensor" command before a physical collision occurs.
</Accordion>
